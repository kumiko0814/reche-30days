# Re:che 30日間プログラム学習ゲーム

毎朝のボイス課題をゲーム感覚で楽しく提出できる学習支援アプリです。Re:che入会後30日の基礎カリキュラムを、スタンプカレンダー・バッジ・レベル称号・フィードバック機能で、圧をかけない優しいトーン設計で進めます。

## 目的

Re:che入会者の**提出率向上**と**学習継続モチベーション維持**。毎日1つの課題（テキスト提出・クイズ・計算機・月次レポート）に取り組み、進捗は自動で管理されます。

## ファイル構成

| ファイル | 役割 |
|---------|------|
| `curriculum.js` | 30日間の課題定義（Day1-30）、バッジ、レベル定義 |
| `kumiko-messages.js` | くみこから毎日の応援メッセージ（30日分）＋画像/音声の設定 |
| `app-core.js` | 進捗保存・採点・レベル算出・連続ボーナス・ランキング・フィードバック機能の核 |
| `styles.css` | 女性向け可愛いテイスト（Zen Maru Gothic、テラコッタ×フォレスト色） |
| `index.html` | ユーザー版アプリ（本人の進捗確認・課題提出） |
| `admin.html` | 管理者版アプリ（全メンバー状況把握・フィードバック入力） |
| `assets/` | くみこの画像 `kumiko.png` と任意の音声を置く場所（手順は assets/README.txt） |

## 追加のゲーム機能

- **くみこから毎日の応援メッセージ**：ホームと各Dayモーダルに、その日のテーマに沿ったメッセージを表示。`assets/kumiko.png` を置くと顔写真付きに（無い間は「く」アイコンを自動表示）。`kumiko-messages.js` の `audio` にURLを入れるとボイス再生ボタンも出せます。
- **連続提出ボーナス**：Day1から連続提出するほどボーナスポイント（3日+50 / 7日+150 / 14日+300 / 21日+500 / 30日+800）。ヒーローに「あと◯日でボーナス」を表示。
- **同期ランキング**：同期メンバーをポイント順に表示し、自分の順位を強調。健全な励まし合いを後押し（実データでもランキングが効果的でした）。
- メッセージ文面は `kumiko-messages.js` を直接編集すれば差し替え可能です。

## 使い方

### ローカル環境で試す

1. このフォルダの `index.html` をブラウザで開く → ユーザー版が起動
2. ニックネームを入力して「スタート」 → Day 1 から課題が解放される
3. 毎日1課題を提出 → ポイント加算・バッジ獲得・レベルが自動更新
4. `admin.html` をブラウザで開く → 管理者版が起動

**初回起動時は、デモメンバー4名（ゆうこ・のりこ・まゆみ・さおり）が自動表示されます。**

### 管理者版の機能

- メンバー一覧（ニックネーム・進捗日数・ポイント・レベル）
- 個別メンバーの詳細進捗確認
- Day ごとのフィードバック入力（例："目的が明確で素敵です！"）
- JSON形式でのデータインポート（ユーザー版から「進捗を送る」で書き出されたファイル）

## ゲーム設計

### ポイントシステム

各課題をクリアするとポイント獲得。基礎ポイント＋ボーナス：

- **基礎ポイント**: Day ごとに異なる（100～300点）
- **クイズボーナス**: 1問正答＝+20点、全問正解＝+50点追加
- **累積ポイント**: Day 1～30 を全クリアで約4,500点

### レベル（リッシュランク）と称号

進捗に応じて自動ランクアップ：

| Lv | 称号 | 必要ポイント |
|--:|------|------------|
| 1 | かけだしセラー | 0 |
| 2 | コツコツセラー | 400 |
| 3 | リサーチャー | 900 |
| 4 | 出品マイスター | 1,600 |
| 5 | 数字マスター | 2,400 |
| 6 | Re:che セラー | 3,400 |

### バッジ（全11個）

課題達成時に自動獲得：

- Day 3: 「はじめの一歩」（目標設定完了）
- Day 4: 「数字に強い」（クイズ挑戦）
- Day 7: 「お店オープン」（プロフィール完成）
- Day 10: 「スピードアップ」（1時間出品チャレンジ）
- Day 12: 「在庫マスター」（在庫管理表提出）
- Day 14: 「折り返し地点」（中間報告）
- Day 18: 「出品の型」（商品ページ添削）
- Day 19: 「出品WEEK突入」（30品チャレンジ開始）
- Day 25: 「30品チャレンジ完走」（1週間走り切った）
- Day 28: 「1ヶ月完走」（月報提出）
- Day 30: 「Re:che 30DAYS 完走」（卒業）

### スタンプカレンダー

Day ごとの提出状況を視覚化（10日×3段）。完了すると「済」スタンプが付く。連続提出日数（ストリーク）も表示。

### 課題タイプ

| type | 説明 | 例 |
|------|------|-----|
| `submit` | テキスト/URL提出 | ライフスタイル書き出し、プロフィールURL、感想提出 |
| `quiz` | クイズ自動採点 | Day 4 販売率・利益率クイズ（3問） |
| `calc` | 計算機（自動出力） | Day 6 「必要な出品数」逆算計算 |
| `report` | 月次レポート | Day 14 中間報告、Day 28 月報 |
| `action` | チェックリスト | 仕入れサイト登録、在庫管理表作成 |

## データの流れ

### 現状（Phase 1）

**保存先**: localStorage（端末内のみ）

1. **ユーザー版**: 課題を提出 → `localStorage` に自動保存
2. **管理者版**: メンバー一覧表示（同じ端末内のローカルデータから読込）
3. **データ共有**: ユーザー版の「進捗を送る」→ JSON ダウンロード → 管理者版の「データ取込」で集約

**注意**: クロス端末での自動同期はしません。管理者が手動でデータを集約する仕組みです。

### 本番化ロードマップ（Phase 2）

以下の実装予定：

- **バックエンド連携**: GAS（Google Apps Script）+ Google スプレッドシート、または Firebase
- **リアルタイム集約**: 全メンバーの進捗をサーバーで一元管理
- **LINE連携**: 毎朝のボイス配信とアプリ提出を同期
- **自動フィードバック**: 提出完了後、事務局フィードバックが即座に表示

**実装構造**: `app-core.js` の以下の関数を差し替える設計：
- `publish(s)` → サーバーへのアップロード
- `importRosterJSON(text)` → サーバーからのデータ取得
- `setFeedback(nick, day, text)` → GAS経由でのフィードバック保存

## Netlify での公開手順

このフォルダを静的サイトとして公開できます。

1. Netlify にログイン
2. 「Add new site」 → 「Deploy manually」を選択
3. このフォルダ全体をドラッグ&ドロップ
4. 公開 URL が発行される

**アクセス:**
- ユーザー版: `https://your-site.netlify.app/` （index.html）
- 管理者版: `https://your-site.netlify.app/admin.html`

**セキュリティ**: 管理者版 URL は関係者のみに共有してください。本番環境では認証機能の追加を推奨します。

## カリキュラムの編集方法

`curriculum.js` の `RECHE_CURRICULUM` 配列を編集します。

### 基本構造

```javascript
{
  day: 1,
  stage: "準備編",
  voice: "ようこそRe:cheへ",  // 毎朝のボイスメッセージテーマ
  title: "課題タイトル",
  detail: "250〜300字程度の課題説明",
  type: "submit",              // submit | quiz | calc | report | action
  points: 100,
  badge: "バッジID" または null,
  placeholder: "入力例",
  // type に応じた追加プロパティ
  quiz: [ { q: "問題文", answer: 85, ... } ],  // type=quiz のとき
  calc: { inputs: [...], outputs: [...] },     // type=calc のとき
  report: [ { key: "...", label: "...", ... } ], // type=report のとき
}
```

### 編集後の反映

新規メンバーは編集後の カリキュラムが適用されます。既存メンバーには影響しません。

## トラブルシューティング

### データが保存されない

- ブラウザの localStorage が有効か確認
- プライベートブラウジングモードで開いていないか確認
- キャッシュをクリアして再度試す

### 管理者版でメンバーが表示されない

- ユーザー版で「進捗を送る」で JSON をダウンロード
- 管理者版で「データ取込」にアップロード
- 初回起動なら、デモメンバー 4 名が自動表示されます

### 日付がロックされたまま

- `localStorage` の `reche30_state` の `startDate` を確認
- `mode: "date"` の場合、当日までの課題が解放されます
- `mode: "free"` に変更すると全課題が解放されます（テスト用）
